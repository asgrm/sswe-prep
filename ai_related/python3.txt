47. What does the `with` statement do in Python?
  Automatically handles setup and cleanup of resources (like files, connections, locks).

  syntax:

    with resource as variable:

  Common Examples:
    # Files
    with open('file.txt', 'r') as f:
        data = f.read()
    # File automatically closes

    # Multiple resources
    with open('in.txt', 'r') as infile, open('out.txt', 'w') as outfile:
        outfile.write(infile.read())
    # Both close automatically
    # if the same file opened in "r" as infile and "w" as outfile the content of file would be cleared by "w" first and it wouldn't be possible to read it

    # Locks
    import threading
    lock = threading.Lock()

    with lock:
        # Critical section
        shared_data += 1
    # Lock automatically released

    # db
    import sqlite3

    with sqlite3.connect('database.db') as conn:
        cursor = conn.cursor()
        cursor.execute('SELECT * FROM users')
        results = cursor.fetchall()

    # Connection closes automatically

48. What are async context managers in Python and how do you use them?
  Purpose: Automatically handle setup and cleanup for async resources (network connections, database connections, async files).

  Syntax: `async with` instead of `with`

  example:

  async def fetch_multiple():
    async with aiohttp.ClientSession() as session:
        # Reuse one session for multiple requests
        async with session.get(url1) as resp1:
            data1 = await resp1.json()

        async with session.get(url2) as resp2:
            data2 = await resp2.json()

      # Session closes automatically
        return data1, data2

  OR

  import asyncio
  import aiohttp

  async def fetch_url(url):
      async with aiohttp.ClientSession() as session:
          async with session.get(url) as response:
              return await response.text()

  asyncio.run(fetch_url("https://example.com"))

50. How do you iterate over async operations in Python?
  import asyncio

  async def fetch_item(n):
      await asyncio.sleep(0.1)
      return f"Item {n}"

  async def main():
      # Sequential (one at a time)
      for i in range(3):
          item = await fetch_item(i)
          print(item)

      # Concurrent (all at once)
      tasks = [fetch_item(i) for i in range(3)]
      results = await asyncio.gather(*tasks)
      print(results)  # ['Item 0', 'Item 1', 'Item 2']

  asyncio.run(main())

  # Sequential: Use for loop with await
  # Concurrent: Create list of tasks, use asyncio.gather(*tasks)

  NOTE: *tasks is example of unpack operator, because asyncio.gather takes positional arguments and not list!

55. How do you schedule a regular (non-async) function to run on the next event loop iteration?

56. How do you schedule a callback to run after a delay?

60. How do you wait for tasks with custom completion conditions (unlike gather())?

  Use asyncio.wait() - Wait with control over when to return

    import asyncio

    async def fetch(name, delay):
        await asyncio.sleep(delay)
        return f"Result from {name}"

    async def bad_task(name):
        await asyncio.sleep(1)
        raise ValueError(f"Error in {name}")

    async def main():
        tasks = [
            asyncio.create_task(fetch("A", 3)),
            asyncio.create_task(bad_task("B")),
            asyncio.create_task(fetch("C", 2))
        ]

        done, pending = await asyncio.wait(
            tasks,
            return_when=asyncio.FIRST_EXCEPTION,
            timeout=4 # optional timeout
        )

        for task in done:
          try:
              result = task.result()
              print(f"Success: {result}")
          except Exception as e:
              print(f"Error: {e}")

        for task in pending:
            task.cancel() # it's important to cancel tasks if any

    asyncio.run(main())

  Options:

    FIRST_COMPLETED - Return when first completes
    FIRST_EXCEPTION - Return when first raises exception
    ALL_COMPLETED - Wait for all (default)

  done, pending are sets with complete/pending operation
  asyncio.wait() works only with tasks, not coroutines

  Use when: You need fine control over completion conditions

61. How do you process task results as soon as they're ready (not in original order)?

  Use asyncio.as_completed() - Iterate in completion order

    async def main():
        tasks = [
            task("A", 2),
            task("B", 1),
            task("C", 3)
        ]

        # Process in completion order (B, A, C)
        for coro in asyncio.as_completed(tasks):
            result = await coro
            print(result)  # B done, A done, C done

  Use when: You want to process results as soon as they're ready

63. How do you protect a task from being cancelled in Python? (Can you cancel tasks in Python?)

65. how to shuffle list values do in Python?

  random.shuffle(list) Shuffles a list in place (modifies the original list), returns None (result is stored in original variable)

      import random

      cards = ['A', 'K', 'Q', 'J']
      a = random.shuffle(cards)
      print(cards)  # ['Q', 'A', 'J', 'K'] (random order)
      print(a) # None

  Use it:
    When you want to randomize the entire list in place
    When you don't need the original order
    More memory efficient (no new list created)

  random.sample(population, k) - Returns a list of k unique random elements from the population (without replacement)

      import random

      # Pick 3 random cards from deck
      cards = ['A', 'K', 'Q', 'J', '10', '9']
      hand = random.sample(cards, 3)
      print(hand)    # ['Q', '9', 'A'] (random selection)
      print(cards)   # ['A', 'K', 'Q', 'J', '10', '9'] (unchanged!)

      # Shuffle by sampling all elements
      shuffled = random.sample(cards, len(cards))
      print(shuffled)  # All cards in random order

      # Works with strings
      word = "HELLO"
      random.sample(word, 3)  # ['L', 'H', 'E']

      # Works with ranges
      random.sample(range(100), 5)  # [23, 67, 2, 89, 45]

      # Random lottery numbers (no duplicates)
      lottery = random.sample(range(1, 50), 6)

      # Select random winners
      winners = random.sample(participants, 3)

      # Random subset of data
      subset = random.sample(dataset, 100)

  Memory tip: sample() takes a "sample" of the population, like sampling in statistics - you get a subset without affecting the original!

  Use it:
    When you need to select a subset (k < n)
    When you want to preserve the original
    When working with immutable sequences (tuples, strings)

79. What is collections.deque and when should you use it instead of a list?
  "Double-ended queue" - optimized for fast operations at BOTH ends

  from collections import deque
  d = deque([1, 2, 3, 4, 5])
  d.popleft()       # O(1) - fast removal from beginning
  d.appendleft(0)   # O(1) - fast addition to beginning
  d.extend([6, 7])  # [5, 1, 2, 3, 4, 6, 7] - extend right
  d.extendleft([0]) # [0, 5, 1, 2, 3, 4, 6, 7] - extend left

  l = deque([1, 2, 3, 4, 5])
  l.rotate(-1)  # [2, 3, 4, 5, 1] - rotate left
  OR
  l.rotate(2)       # [4, 5, 1, 2, 3] - rotate right


  d.reverse() - reverse in place like list[::-1]

   Use deque when:
    Implementing queues (FIFO)
    Need frequent operations at both ends
    Building sliding windows
    Implementing breadth-first search (BFS)
    Need a circular buffer

  DO NOT USE when:
    Only need stack operations (append/pop from end)
    Need frequent random access by index
    Need list comprehensions or slicing

116. How does multiple inheritance work in Python? What is MRO?

  class Animal:
      def eat(self):
          print("Eating...")

  class Flyer:
      def fly(self):
          print("Flying...")

  class Bird(Animal, Flyer):  # Inherits from both
      pass

  bird = Bird()
  bird.eat()  # From Animal
  bird.fly()  # From Flyer

  Method Resolution Order (MRO):
    Order matters! Python checks parents left to right.

  class A:
    def sound(self):
        print("A")

  class B:
      def sound(self):
          print("B")

  class C(A, B):  # A is checked first
      pass

  C().sound()  # Output: A

  # Check order:
  print(C.__mro__)  # C -> A -> B -> object


  # Order matters - leftmost parent checked first
  # MRO (Method Resolution Order) determines which method is called
  # Use super() to follow MRO

  List comprehension is a concise way to create lists in Python using a single line of code. List comprehention can use different iterables(sequences) such as lists, strings, ranges and tuples

  syntax:
  [expression for item in iterable if condition]  # "if condition" is optional

  IMPORTANT: "if condition" runs before "expression" so if may also cut out edge cases

158. What are Python’s bitwise operators and what do they do?
  Python bitwise operators operate on integers at the binary level:

    & — bitwise AND

    | — bitwise OR

    ^ — bitwise XOR

    ~ — bitwise NOT

    << — left shift

    >> — right shift

  & | ^  also can operate on sets
  examples:

  5 → 0101
  3 → 0011

  5 & 3 = 0001 = 1
  5 | 3 = 0111 = 7
  5 ^ 3 = 0110 = 6

  5 = 00000101
  ~5 = 11111010 = -6


  5 = 00000101
  5 << 1 =  00001010 = 10
  So, (for non-negative integers) x << n == x * (2 ** n)

  >> -  is opposite
  10 = 00001010
  10 >> 1 = 00000101 = 5
  So, (for non-negative integers) x >> n == x // (2 ** n)

      def __init__(self):
          self.items = []

      def add(self, item, priority):
          insort(self.items, (priority, item))

      def pop(self):
          return self.items.pop(0)[1] if self.items else None

  pq = PriorityQueue()
  pq.add("Low priority", 5)
  pq.add("High priority", 1)
  pq.pop()  # Returns "High priority"

178. Why are Metadata Annotations (typing.Annotated) useful in Python?
  Annotated lets you attach extra metadata to a type hint without changing the actual type.
  Python itself ignores the metadata.
  Editors and type checkers still see the type as str.
  It allows frameworks (e.g. FastAPI) to add behavior using metadata

    from typing import Annotated

    def say_hello(name: Annotated[str, "this is just metadata"]) -> str:
        return f"Hello {name}"

186. How to specify query parameters?

  Any function parameter not included in the path template is treated as a query parameter.

  Query parameters arrive as strings, but are automatically converted and validated using Python type annotations.

  Providing a default value makes a query parameter optional.

  If omitted, FastAPI uses the default value.

  Setting the default to None (e.g. str | None) makes the parameter optional, with value None when missing.

    from fastapi import FastAPI

    app = FastAPI()

    @app.get("/items/")
    async def read_items(skip: int = 0, q: str | None = None):
        return {"skip": skip, "q": q}

187. How does FastAPI handle boolean query parameters?

  Declare them using type bool with a default (e.g. short: bool = False)

  FastAPI automatically converts query values to True or False

  Values interpreted as True include ( Any case variation):  1, true, True, on, yes

  Any other value (or missing parameter) results in False

    from fastapi import FastAPI

    app = FastAPI()

    @app.get("/items/{item_id}")
    async def read_item(item_id: str, short: bool = False):
        if short:
            return {"item_id": item_id}
        return {
            "item_id": item_id,
            "description": "This is a long description"
        }

190. How to parse body from request?

  The function parameters will be recognized as follows:

    If the parameter is also declared in the path, it will be used as a path parameter.
    If the parameter is of a singular type (like int, float, str, bool, etc) OR enum it will be interpreted as a query parameter.
    If the parameter is declared to be of the type of a Pydantic model, it will be interpreted as a request body.

    from fastapi import FastAPI
    from pydantic import BaseModel

        name: str
        description: str | None = None
        price: float
        tax: float | None = None

    @app.put("/items/{item_id}")

        result = {"item_id": item_id, **item.dict()}
        if q:
            result.update({"q": q})
        return result

232. How and why to specify dependencies of the path operation decorator?

  In some cases you don't really need the return value of a dependency inside your path operation function.

  Or the dependency doesn't return a value.

  But you still need it to be executed/solved.

  For those cases, instead of declaring a path operation function parameter with Depends, you can add a list of dependencies to the path operation decorator.

  Example:

    from typing import Annotated

    from fastapi import Depends, FastAPI, Header, HTTPException

    app = FastAPI()

    # conditionally raise the exception
    async def verify_token(x_token: Annotated[str, Header()]):
        if x_token != "fake-super-secret-token":
            raise HTTPException(status_code=400, detail="X-Token header invalid")

    # conditionally raise the exception
    async def verify_key(x_key: Annotated[str, Header()]):
        if x_key != "fake-super-secret-key":
            raise HTTPException(status_code=400, detail="X-Key header invalid")
        # returned value is discarded
        return x_key

    # each item of dependencies list should be wrapped into "Depends"

    @app.get("/items/", dependencies=[Depends(verify_token), Depends(verify_key)])
    async def read_items():
        return [{"item": "Foo"}, {"item": "Bar"}]

234. How and why use dependencies with yield?

  async def get_db():
      db = DBSession()
      try:
          yield db
      finally:
          db.close()

  The yielded value is what is injected into path operations and other dependencies.

  The code following the yield statement is executed after the response.

  code before yield executed -> yield executed and injected -> request finished -> code in dependency after yield executed

  If some code at some point in the middle, in another dependency or in a path operation, made a database transaction "rollback" or created any other exception, you would receive the exception in your dependency.

  So, you can look for that specific exception inside the dependency with except SomeException.

  In the same way, you can use finally to make sure the exit steps are executed, no matter if there was an exception or not.

235. Explain flow of execution if multiple (3) dependency depends on each other (subdependencies)?

  if path operation function depends on `dep_c` which depends on `dep_b` which depends on `dep_a` the order is following:

  pre yield + yield of dep_a
  pre yield + yield of dep_b
  pre yield + yield of dep_c

  path operation function

  finally of dep_c
  finally of dep_b
  finally of dep_a

236. What are dependencies scope?
  In Depends by default scope="request" is used, the only other option is scope="function".

  Order of execution:
    for "request": yield of dependency -> path operastion function -> response send to the client -> finally of dependency
    for "function": yield of dependency -> path operastion function -> finally of dependency -> response send to the client

  If dependency has scope "request" all subdependencies should be of scope "request" as well to be executed in proper order.
  If dependency has scope "function" dependencies may have  "request" | "function" scope

Note: When you create a dependency with yield, FastAPI will internally create a context manager for it, and combine it with some other related tools.

Note: In Python, you can create Context Managers by creating a class with two methods: __enter__() and __exit__().  Another way to create a context manager is with python's @contextlib.contextmanager or @contextlib.asynccontextmanager

244. What are BackgroundTasks in FastAPI and when should you use them?
  BackgroundTasks let you run functions after returning a response. Use them for non-critical, post-response work (e.g., sending emails, logging, light processing) so the client doesn’t wait. Tasks can be def or async def. Example:

    from fastapi import BackgroundTasks, FastAPI

    app = FastAPI()

    def write_notification(email: str, message=""):
        with open("log.txt", mode="w") as email_file:
            content = f"notification for {email}: {message}"
            email_file.write(content)

    @app.post("/send-notification/{email}")
    async def send_notification(email: str, background_tasks: BackgroundTasks):
        background_tasks.add_task(write_notification, email, message="some notification")
        return {"message": "Notification sent in the background"}

245. How do BackgroundTasks work with FastAPI dependencies?

  BackgroundTasks can be injected into path operations, dependencies, and sub-dependencies. FastAPI reuses the same instance, merges all added tasks, and executes them together after the response is sent. Example:

    from typing import Annotated

    from fastapi import BackgroundTasks, Depends, FastAPI

    app = FastAPI()

    def write_log(message: str):
        with open("log.txt", mode="a") as log:
            log.write(message)

    def get_query(background_tasks: BackgroundTasks, q: str | None = None):
        if q:
            message = f"found query: {q}\n"
            background_tasks.add_task(write_log, message)
        return q

    @app.post("/send-notification/{email}")
    async def send_notification(
        email: str, background_tasks: BackgroundTasks, q: Annotated[str, Depends(get_query)]
    ):
        message = f"message to {email}\n"
        background_tasks.add_task(write_log, message)
        return {"message": "Message sent"}

257. How do you make a Python class that supports custom context management, both sync (with) and async (async with)? What methods are required and when are they used?

258. What is io.BytesIO in Python? When should you use it, and how does it behave compared to files on disk?

  io.BytesIO is an in-memory binary stream that behaves like a file opened in binary mode (rb / wb).

    from io import BytesIO

    with open(image_path, "rb") as f:
        #Reads an image from disk as raw bytes
        image_bytes = f.read()
    # Wraps those bytes in a file-like object, Acts like a real binary file in memory
    image_content = BytesIO(image_bytes)


  Many libraries (e.g. httpx, requests, PIL, zipfile) expect an object that:

  - Has .read(), .seek(), .tell()
  - Behaves like an open file
  - They cannot work with raw bytes alone.

  Example (HTTP upload):

  files = {
      "image": ("banner.png", image_content, "image/png")
  }

  Why not pass bytes directly?
    bytes != file; bytes has no cursor, no .read(), no .seek(); File-based APIs break without a file-like interface

  If you write to a BytesIO, you must rewind before reading:
    buf = BytesIO()
    buf.write(image_bytes)
    buf.seek(0)  # REQUIRED

  This is not needed when using:
    BytesIO(image_bytes)
